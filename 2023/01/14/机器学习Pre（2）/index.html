Improving Deep Neural Networks Hyperparameter tuning, Regularization and Optimization

Train/dev/test sets

  • train set：训练集
  • devlopment set：验证集
  • test set：测试集

验证集为训练集和测试集所重合的部分

对于少量的数据，可以采取70%训练集、30%测试集的方法，或者80%训练集、20%验证集、20%测试集的方法。对于大量的数据，例如百万级的数据，测试集和验证集所占的比重将远小于20%

Bias and Variance

variance指方差，bias指偏差

variance描述的是数据偏离期望值的程度，而偏差描述预测值偏离实际值的程度。也就是说variance描述的是数据本身的性质，而bias描述的是数据和人的期望之间的差距

对于high bias的情况，我们称为underfitting（欠拟合），对于high variance的情况，我们称为过拟合

假设Optimal （Base） error接近于0，则如果出现下列情况

  • Train set error很小、Dev set error很大：则出现high variance，模型过拟合
  • Train set error很大：则出现high bias，模型欠拟合
  • 都很大：可能是因为部分的过拟合了数据

Why regularization reduce overfitting

直观上来说，随着W的值不断减小，如果减小为0，那么对应的神经元就无效了。因此整个神经网络就变成了一个更简单的神经网络模型。尽管事实上很难发生这种情况，但是随着W的值的减少，大量的神经元的影响也随之变小，而越来越贴近简单的logistics regression，从而避免过拟合。

此外，对于一些非线性的激活函数，例如tanh，当W变小，Z=WTA+b也会随着减小。而在趋于0的部分tanh函数将会趋向于简单的线性函数。而对于一个线性函数而言，无论多少次拟合，线性函数之间的拟合最终都会是线性函数，不会拟合出复杂的非线性函数，防止模型的过拟合

Dropout regularization

Dropout regularization会遍历神经网络中的每个节点，设置每个节点保留和消除的概率。在设置完概率之后，对于每个训练样本，都会依概率消除掉一些节点，并删除与这些节点相连的线。因此对于每个训练样本，都会采取一个不同的、精简的神经网络模型去进行训练。

Optimize algorithm

Mini-batch gradient descent

当数据集的范围过于巨大。尽管使用向量化加速了运算，但实现运算和梯度下降的时间仍然很长。而这个时候我们可以通过先进行部分的梯度训练来减少最终训练所需的时间。

通过取整个训练集的几个较小的子集，被称为mini-batch，例如将mini-batch的大小设为1000，则可以取x(1),x(2),...,x(1000)作第一个为mini-batch，x(1001),x(1002),...,x(2000)作为第二个mini-batch...并将前1000个训练样本组成的集合设为X{1}，随后的1000个样本设为X{2}。对应的，则有Y{1},Y{2},...而mini-batch t则表示X{t},Y{t}

对各个mini- batch进行梯度下降训练即可，也就是对于每个mini-batch进行以下步骤，直至完成所有的mini-batch的训练，或者是达到所需要的精度

  • forward propogation
  • Compute the cost of mini-batch
  • backward propogation
  • update the weight

每完成一次这样的步骤则被称为完成了一个epoch

和mini-batch gradient descent相比，Batch gradient descent对训练集中的所有样本进行训练，容易看出每次迭代损失函数的值都会减少，也就是说损失函数相对于迭代次数是一个单调递减的函数。而mini-batch由于每次只对训练集的不同子集进行训练，因此损失函数会出现波动，但整体的趋势应当是减少。

在实现mini-batch gradient descent时，需要决定的是每个batch的大小

如果batch-size为m，则mini-batch gradient descent就化为了batch gradient descent

如果batch-size为1，则mini-batch gradient descent就化为了stochastic gradient descent
